Authentication

Every request must include your API key. Pass it in the HTTP header or as a query-string parameter.

Header (recommended)
X-API-Key: YOUR_API_KEY
Query parameter (fallback)
GET /api/posts?api_key=YOUR_API_KEY
API keys are issued by administrators. Contact your admin or visit your profile to request one.
Generate a Key Admin only
POST/api/generate_key

JSON body:

{
  "user_id": 42,
  "permissions": "post:create"
}

Response 201:

{
  "api_key": "abc123...",
  "user_id": 42,
  "permissions": "post:create"
}

Posts

POST /api/posts Create a new post

Send as multipart/form-data so you can attach image and/or video files.

FieldTypeRequiredDescription
titlestringyesPost title
contentstring (HTML)yesPost body – HTML tags are sanitised server-side
category_idintegernoCategory by numeric ID
category_namestringnoCategory by name — auto-creates the category if it doesn't exist yet
imagefilenoFeatured image: png jpg jpeg gif webp
videofilenoEmbedded video: mp4 webm mov avi mkv
video_urlstringnoFilename of an already-uploaded video (alternative to uploading)

Response 201:

{
  "message": "Post created successfully",
  "post_id": 101,
  "post_url": "https://example.com/see_more/101",
  "has_video": true,
  "category": { "id": 3, "name": "Technology" }
}
GET /api/posts List your posts
ParameterDefaultDescription
page1Page number
per_page10Results per page (max 50)
category_id–Filter by category ID
category–Filter by partial category name (case-insensitive)
has_video–Pass 1 to return only posts with an attached video

Response 200:

{
  "posts": [
    {
      "id": 101,

      "video_url": null,
      "image_url": "photo.jpg",
      "category": { "id": 3, "name": "Technology" },
      "url": "https://example.com/see_more/101"
    }
  ],
  "total": 45,
  "pages": 5,
  "current_page": 1,
  "has_next": true,
  "has_prev": false
}
GET /api/posts/<post_id> Get a single post

Returns the full post object including content (HTML), video_url, and has_video.

The post must belong to the authenticated API user.

Videos

The Videos resource lets you upload standalone videos (separate from posts). Supported formats: mp4 webm mov avi mkv.

POST /api/videos Upload a video

Send as multipart/form-data.

FieldTypeRequiredDescription
titlestringyesVideo title
descriptionstringyesShort description
videofileyesVideo file: mp4 webm mov avi mkv

Response 201:

{
  "message": "Video uploaded successfully",
  "video_id": 7,
  "video_url": "my-clip.mp4"
}
GET /api/videos List your videos

Supports page and per_page (max 50) query parameters.

{
  "videos": [
    {
      "id": 7,
      "title": "Product Demo",
      "description": "Watch how it works",
      "video_url": "demo.mp4",
      "uploaded_at": "2025-03-01T12:00:00"
    }
  ],
  "total": 3,
  "pages": 1,
  "current_page": 1,
  "has_next": false,
  "has_prev": false
}
DELETE /api/videos/<video_id> Delete a video

Deletes both the database record and the physical file. The video must belong to the authenticated user.

{ "message": "Video deleted successfully" }

Categories

Categories are hierarchical (parent → children). You can reference them by ID or by name; when creating a post with category_name, the category is auto-created if it doesn't already exist.

GET /api/categories List all categories

Default returns a tree structure. Add ?flat=1 for a simple list.

Tree (default)

{
  "categories": [
    {
      "id": 1,
      "name": "Technology",
      "parent_id": null,
      "children": [
        { "id": 4, "name": "AI", "parent_id": 1, "children": [] }
      ]
    }
  ],
  "total": 5
}

Flat ?flat=1

{
  "categories": [
    { "id": 4, "name": "AI", "parent_id": 1 },
    { "id": 1, "name": "Technology", "parent_id": null }
  ],
  "total": 5
}
POST /api/categories Create a category

Send JSON. If the category name already exists the existing record is returned (idempotent).

// Request body
{ "name": "Machine Learning", "parent_id": 1 }

// Response 201 (new) or 200 (already exists)
{
  "message": "Category created successfully",
  "category": { "id": 9, "name": "Machine Learning", "parent_id": 1 }
}
GET /api/categories/<category_id>/posts Posts in a category

Returns paginated public posts for any category (not just your own). Supports page and per_page.

{
  "category": { "id": 1, "name": "Technology" },
  "posts": [
    {
      "id": 101,
      "title": "AI Trends 2025",
      "excerpt": "...",
      "published_date": "2025-03-01T10:00:00",
      "author": "alice",
      "views": 412,
      "likes": 30,
      "has_video": true,
      "url": "https://example.com/see_more/101"
    }
  ],
  "total": 12, "pages": 2, "current_page": 1,
  "has_next": true, "has_prev": false
}

Error Reference

HTTP StatusMeaningCommon cause
400Bad RequestMissing required fields, invalid file format
401UnauthorisedMissing or invalid API key
403ForbiddenAPI key lacks required permissions
404Not FoundPost / video / category does not exist or belongs to another user
500Server ErrorUnexpected internal error

All error responses follow the same shape: { "error": "Description of the problem" }

Examples

curl -X POST \
  -H "X-API-Key: YOUR_API_KEY" \
  -F "title=My New Article" \
  -F "content=<p>This is the body.</p>" \
  -F "category_name=Technology" \
  -F "image=@/path/to/photo.jpg" \
  {{ url_for('api.create_post', _external=True) }}
curl -X POST \
  -H "X-API-Key: YOUR_API_KEY" \
  -F "title=Video Report" \
  -F "content=<p>Watch the clip below.</p>" \
  -F "category_id=3" \
  -F "video=@/path/to/clip.mp4" \
  {{ url_for('api.create_post', _external=True) }}
# Upload a standalone video
curl -X POST \
  -H "X-API-Key: YOUR_API_KEY" \
  -F "title=Product Demo" \
  -F "description=Watch how the feature works" \
  -F "video=@/path/to/demo.mp4" \
  {{ url_for('api.create_video', _external=True) }}

# List your videos
curl -H "X-API-Key: YOUR_API_KEY" \
  "{{ url_for('api.get_videos', _external=True) }}?page=1&per_page=10"

# Delete a video
curl -X DELETE \
  -H "X-API-Key: YOUR_API_KEY" \
  {{ request.host_url }}api/videos/7
# Get hierarchical tree
curl -H "X-API-Key: YOUR_API_KEY" \
  {{ url_for('api.get_categories', _external=True) }}

# Get flat list
curl -H "X-API-Key: YOUR_API_KEY" \
  "{{ url_for('api.get_categories', _external=True) }}?flat=1"

# Create a sub-category
curl -X POST \
  -H "X-API-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"name": "Machine Learning", "parent_id": 1}' \
  {{ url_for('api.create_category', _external=True) }}

# Get all posts in category 3
curl -H "X-API-Key: YOUR_API_KEY" \
  {{ request.host_url }}api/categories/3/posts
import requests

API_KEY = "YOUR_API_KEY"
BASE    = "{{ request.host_url }}"
HEADERS = {"X-API-Key": API_KEY}

# ── Create a text-only post ──────────────────────────────────────
r = requests.post(BASE + "api/posts", headers=HEADERS, data={
    "title": "Hello World",
    "content": "<p>First API post!</p>",
    "category_name": "General",   # auto-created if absent
})
print(r.json())

# ── Create a post with a video ───────────────────────────────────
with open("clip.mp4", "rb") as vid:
    r = requests.post(BASE + "api/posts", headers=HEADERS, data={
        "title": "Video Post",
        "content": "<p>See the video below.</p>",
        "category_id": 3,
    }, files={"video": vid})
print(r.json())

# ── Upload a standalone video ────────────────────────────────────
with open("demo.mp4", "rb") as vid:
    r = requests.post(BASE + "api/videos", headers=HEADERS, data={
        "title": "Product Demo",
        "description": "A short walkthrough",
    }, files={"video": vid})
print(r.json())

# ── Filter posts with video in a specific category ───────────────
r = requests.get(BASE + "api/posts", headers=HEADERS,
                 params={"category": "Technology", "has_video": "1"})
for post in r.json()["posts"]:
    print(post["title"], post["url"])
